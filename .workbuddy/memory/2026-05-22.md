## SSE 真实进度条改造

### 问题
前端三个生成流程（大纲/标题/正文）的进度条是假的——纯前端 setInterval 匀速动画，与后端 Agent 实际执行进度无关。

### 解决方案
实现 SSE（Server-Sent Events）实时推送，后端在每个 Agent 步骤开始/完成时推送事件，前端根据真实事件驱动进度条。

### 后端改动
1. **新增** `backend/app/core/progress.py` — ProgressStore 单例类（内存队列 + async generator SSE 流）
2. **改造** `backend/app/services/outline_generation/outline_service.py` — generate_outline() 新增 progress_callback 参数
3. **改造** `backend/app/services/title_generation_service.py` — TitleGenerationService 新增 progress_callback 参数
4. **改造** `backend/app/services/content_generation/orchestrator.py` — generate_content() 新增 progress_callback 参数
5. **改造** `backend/app/api/v1/outlines.py` — POST /generate 返回 run_id + 新增 GET /stream/{run_id} SSE 端点
6. **改造** `backend/app/api/v1/title_generation.py` — POST / 返回 run_id + 新增 GET /stream/{run_id} SSE 端点
7. **改造** `backend/app/api/v1/content_generation.py` — POST /generate 改为 asyncio.create_task + SSE

### 前端改动
1. **新增** `frontend/src/composables/useAgentProgress.js` — SSE composable（EventSource + 平滑进度动画）
2. **改造** `frontend/src/components/creation/OutlinePanel.vue` — 删除 simulateProgress，改用 useAgentProgress
3. **改造** `frontend/src/components/creation/TitlePanel.vue` — 同上
4. **改造** `frontend/src/components/creation/ContentPanel.vue` — 同上

### 事件格式
- step_start: {step, agent, action} — Agent 开始工作
- step_done: {step, agent} — Agent 完成工作
- complete: {step, agent} — 全部完成
- result: {...} — 最终结果数据
- error: {message} — 错误

### 注意事项
- EventSource 不支持自定义 header，token 通过 query param 传递
- 前端步骤内使用平滑动画（每 200ms 递增到 90% 封顶），step_done 时跳到 100%
- ProgressStore 使用 asyncio.Queue + TTL 自动清理

### Bug 修复：SSE 流在 result 之前关闭
**问题**：stream 在收到 `complete` 事件后立即关闭，但 `result` 事件在 `complete` 之后推送，导致前端永远收不到结果数据。
**修复**：
1. `progress.py` stream 退出条件从 `complete/error` 改为 `result/error`
2. `useAgentProgress.js` `complete` 只更新进度条，`result` 才关闭 EventSource
3. `title_generation.py` 补推 `result` 事件
