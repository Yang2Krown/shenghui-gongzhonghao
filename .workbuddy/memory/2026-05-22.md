## 正文生成模块设计与实现

### 完成内容
基于《正文生成 Agent 设计文档 v1.1》和《去 AI 味检查清单 v1.0》，设计并实现了 4 Agent 协作的正文生成模块。

### 新增文件
1. **服务模块** (`backend/app/services/content_generation/`)：
   - `__init__.py` - 模块入口
   - `schemas.py` - Pydantic I/O 契约（所有 Agent 输入输出）
   - `orchestrator.py` - 编排器（Agent A → B → C → D）
   - `agent_a_writer.py` - 正文创作员（整篇生成 2500-3000 字）
   - `agent_b_gold_sentence.py` - 金句催化员（3-5 个金句）
   - `agent_c_deai.py` - 去 AI 味改写员（内嵌完整检查清单）
   - `agent_d_inspector.py` - 整合+自检诊断员（8 维度评分）

2. **Celery 任务** (`backend/app/tasks/content_generation_tasks.py`)：
   - `generate_article_task` - 异步正文生成任务

3. **API 端点** (`backend/app/api/v1/content_generation.py`)：
   - `POST /generate` - 异步生成（Celery 任务）
   - `POST /generate/sync` - 同步生成（调试用）
   - `GET /task/{task_id}` - 查询任务状态

4. **设计文档** (`docs/content_generation_design.md`)：
   - 完整的模块设计说明

### 架构特点
- 与选题挖掘模块完全独立，不干扰现有选题 pipeline
- 4 Agent 串行协作：正文创作 → 金句催化 → 去 AI 味 → 整合自检
- 内嵌《去 AI 味检查清单》6 大类 30+ 表达模式
- 8 维度自检评分（标题兑现、大纲对应、字数、风格、去 AI 味、金句、开头、结尾）
- 不触发重生，决策权 100% 在人工

### LLM 客户端适配
所有 Agent 使用项目统一的 LLM 客户端接口：
- `get_llm_client()` 获取客户端
- `ChatMessage` 构建消息
- `parse_json_loose()` 解析 JSON 输出
- `json_mode=True` 启用 JSON 模式
